🔒 Security Architecture

HIPAA-compliant, defense-in-depth security designed to protect sensitive health data from all attack vectors

⚠️ Threat Model

🔓
Critical
Unauthorized Data Access
Attacker gains access to patient health records including pregnancy data, medications, and medical history.
Mitigation
JWT auth + Row Level Security + AES-256 encryption at rest
💉
Critical
SQL Injection
Attacker injects malicious SQL through input fields to extract or modify database contents.
Mitigation
Prisma ORM with parameterized queries + input validation
🔑
High
Token Theft / Replay
Attacker steals JWT tokens and replays them to gain unauthorized access.
Mitigation
15-min expiry + single-use refresh tokens + token family tracking
🌐
High
Cross-Site Scripting (XSS)
Attacker injects malicious scripts that execute in other users' browsers.
Mitigation
Content Security Policy + output encoding + Helmet.js
🚫
Medium
Broken Access Control
User accesses another user's pregnancy data or caregiver sees unauthorized information.
Mitigation
Row Level Security + ownership verification on every request
⚡
Medium
Brute Force / DDoS
Attacker floods the API with requests or attempts to guess passwords.
Mitigation
Rate limiting + account lockout + CAPTCHA after failures

🛡️ Security Controls

🔐
Authentication
JWT with 15-min expiry, single-use refresh tokens, MFA support, biometric login on mobile
🔒
Encryption
AES-256 at rest, TLS 1.3 in transit, client-side encryption for sensitive data, key rotation
📝
Audit Logging
Every data access logged with user, action, timestamp, IP. 7-year retention. Anomaly detection.
🚫
Input Validation
Class-validator on all DTOs, whitelist strict mode, type checking, length limits
👥
Row Level Security
Users can only access their own data. Caregiver access requires explicit patient consent.
⏱️
Rate Limiting
Per-user rate limits, sliding window, exponential backoff on failures, IP-based throttling
